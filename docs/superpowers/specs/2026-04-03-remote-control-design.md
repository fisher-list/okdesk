# okdesk 设计规格说明书 (Spec)

## 1. 项目背景
开发一款名为 **okdesk** 的个人自用、跨平台（Windows/macOS）远程控制软件。核心目标是实现高性能、低延迟（< 1s）且能穿透复杂防火墙的远程支持与控制。

## 2. 核心需求
- **多显示器支持**：支持多路采集，采用 **B 方案 (标签页/快捷键切换)**。控制端显示一个主屏幕，支持通过顶部菜单或快捷键迅速在多个屏幕间切换，确保单屏显示清晰度。
- **连接管理**：
  - **临时密码模式**：客户端启动后自动生成随机 6 位密码（如 `ok-XXXXXX`），每次启动或手动刷新后变更。
  - **永久密码模式**：用户可手动设置固定密码（如 `MyStrongPass123`），支持无人值守连接。
  - **申请模式**：发起端通过 ID 发起申请，客户端弹出确认框，点击“允许”后开始控制。
- **远程控制**：完全控制远程主机的键盘和鼠标。
- **跨平台支持**：支持 Windows 和 macOS。
- **网络穿透**：必须能穿透防火墙，支持 P2P 直连及中继模式。
- **粘贴板共享**：双向同步系统剪贴板（文本）。
- **文件传输**：支持通过 **文件管理器 (双栏布局)** 进行互传。支持多级目录选择及大文件断点续传，确保传输稳定性。
- **性能指标**：画面清晰流畅，端到端延迟 < 1s。

## 3. 技术架构 (方案一：原生 Rust + WebRTC)

### 3.1 核心组件
- **客户端 (Client)**：
  - **屏幕采集**：
    - Windows: DXGI (Desktop Duplication API)
    - macOS: ScreenCaptureKit
  - **视频编码**：硬件加速 H.264/H.265 (Windows: NVENC/QuickSync; macOS: VideoToolbox)
  - **指令执行**：Windows (SendInput API); macOS (CoreGraphics)
- **控制端 (Controller)**：
  - **视频解码**：硬件加速解码 (FFmpeg/Vulkan/D3D11)
  - **事件捕获**：监听并封装本地键鼠事件通过 DataChannel 发送。
- **信令服务器 (Signaling Server)**：负责交换 ICE Candidate 和连接码 ID 映射。
- **中继服务器 (Relay Server)**：
  - **类型**：用户已安装的中继服务器。
  - **协议**：支持 STUN/TURN (WebRTC 标准)。

### 3.2 数据流
1. **控制流 (Signaling)**：通过 WebSocket 或 HTTP 与信令服务器通信，交换 ID 和 SDP 信息。
2. **媒体流 (Media)**：通过 WebRTC (DTLS/SRTP 加密) 传输视频。
3. **数据流 (Data)**：通过 WebRTC DataChannel 传输键鼠指令、粘贴板数据和文件。

## 4. 安全设计
- **传输加密**：WebRTC 强制执行 DTLS (数据加密) 和 SRTP (媒体加密)。
- **认证机制**：基于随机生成的临时密码进行挑战响应认证。

## 5. 阶段性路线图
1. **阶段 1**：基于 Rust 建立基础的 WebRTC 握手及 P2P 连接（集成已安装中继服务器）。
2. **阶段 2**：实现跨平台的屏幕高速采集与硬件编码传输。
3. **阶段 3**：实现键鼠同步、粘贴板同步及文件传输功能。
4. **阶段 4**：UI 界面打磨与多显示器支持优化。
